## 2024-05-17 - Missing Input Validation on API Routes | **Learning:** API routes like `/api/invitations` often rely on manual destructuring of `request.json()` and simple existency checks rather than comprehensive validation schemas. The centralized `validateInvitationData` function from `@/lib/security` was implemented but not utilized on these endpoints, allowing potentially dangerous inputs (like XSS vectors in descriptions) to bypass frontend controls and reach the database. | **Prevention:** Always pipe `request.json()` payloads through established security validation utilities (e.g., `validateInvitationData` or `validateRequestBody`) immediately after parsing. Only destructure and use properties from the validated, `sanitizedData` object, rejecting requests where `!validation.isValid`.
