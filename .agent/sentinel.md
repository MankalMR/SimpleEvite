## 2024-03-15 - Unauthenticated RLS Bypass in Public RSVP Endpoint
**Learning:** The public `/api/rsvp` endpoint was previously using the anonymous `supabase` client to verify the invitation and insert the RSVP. Because guests are unauthenticated, they lacked the necessary Row Level Security (RLS) privileges to insert into the `rsvps` table, causing the RSVP creation to fail.
**Prevention:** All database interactions for unauthenticated guest flows (like RSVPs) must strictly use the `supabaseAdmin` service role client encapsulated within the `src/lib/database-supabase.ts` Data Access Layer, combined with robust input validation to safely bypass RLS.
